# UPGRADES.md — The earn-it ladder

When to buy the gear I already want. Changes to this file only via rig-change PR, same as `RIG.md`.

## Why this file exists

My known failure mode is spending money to *feel* like progress instead of making progress. This file exists so that "I feel ready" can never be the trigger — a **committed milestone** is. Each upgrade stays locked until the wall it solves is the thing actually blocking my next milestone.

## The principle

> Buy the gear the moment its specific benefit becomes the thing blocking my next committed milestone — proven by evidence already in the repo — and not one milestone sooner.

None of the three upgrades below is on the critical path for phases 1–3. The Taylor GS Mini + numpad + iPad Air 2 (Profile B, see `RIG.md`) can carry the whole way through Phase 4. So every one of these is a **portability / performance** unlock, not a **capability** unlock — which is exactly why they can wait.

## Two traps to guard against

- **Phase 3 power draw is not a premature-iPhone excuse.** When the Akai joins the chain (Phase 3), test iPad + a cheap powered USB-C hub *first* (~CHF 30). Only if that genuinely fails does power become a real argument — and even then, a hub beats a phone.
- **The electric build and the iPhone share the same gate (Phase 4), deliberately.** Shipping Phase 4 is the single "this is really happening now" moment that earns both big purchases at once — one milestone, not three vague vibes.
